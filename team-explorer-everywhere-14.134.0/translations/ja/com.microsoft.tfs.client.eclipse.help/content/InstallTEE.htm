Team Explorer Everywhere プラグインとコマンドライン クライアントのインストール

Eclipse での開発プロセスの一部として Team Explorer Everywhere と Visual Studio Team Foundation Server を使って、作業項目を追跡し、バージョン管理を行い、ビルドを自動化することができます。それらの機能を使うには、Eclipse 用の Team Explorer Everywhere プラグインか Team Foundation Server のクロスプラットフォームのコマンドライン クライアントのいずれかをインストールする必要があります。

Eclipse 用の Team Explorer Everywhere プラグインをインストールする

プラグインは、インストールしてからでなければ開発プロセスに使用できません。 更新サイトから直接 Eclipse にインストールすることも、Eclipse 更新サイト アーカイブからインストールすることもできます。

更新サイトからプラグインをインストールする方法

  1. Eclipse を開きます。
  2. [ヘルプ] メニューで、[新しいソフトウェアのインストール] を選択します。
  3. [インストール] ダイアログ ボックスが表示されます。
  4. [追加] をクリックします。
  5. [リポジトリの追加] ダイアログ ボックスが表示されます。
  6. 名前については、"Team Explorer Everywhere" と入力します
  7. 更新サイトの場所 http://dl.microsoft.com/eclipse/tfs を指定します。
  8. [OK] をクリックします。
  9. [インストール] ダイアログ ボックスにある機能の一覧で、Eclipse 用の Team Explorer Everywhere プラグインに相当するチェック ボックスをクリックします。このオプションが表示されない場合は、[Work with] プルダウン メニューを使って、直前に一覧で入力した更新サイト URL を見つけ、それを選択します。次に、上記のプラグインの隣にあるチェック ボックスを選択します。
  10. [次へ] を 2 回クリックします。
  11. マイクロソフト ソフトウェア ライセンス条項に同意する場合は、同意してから [完了] をクリックします。
  12. 要求されたときに Eclipse を再開します。

更新サイトのアーカイブからプラグインをインストールする方法

    メモ
    上で説明した標準の更新サイトのメカニズムは、推奨されるインストール方法です。ただし、これにはインターネットへの接続が必要であり、Microsoft から最新バージョンのプラグインがインストールされることになります。 インストールするコンピューターが直接インターネットに接続されていない場合、または組織内の全員が特定のバージョンのプラグインをインストールすることが推奨される場合は、次の方法がより適切です。

  1. このページのリンクを使用して TFSEclipsePlugin-UpdateSiteArchive-14.0.4.zip をダウンロードします。
  2. Eclipse を開きます。
  3. [ヘルプ] メニューで、[新しいソフトウェアのインストール] を選択します。
  4. [インストール] ダイアログ ボックスが表示されます。
  5. [追加] をクリックします。
  6. [リポジトリの追加] ダイアログ ボックスが表示されます。
  7. 名前については、"ローカル Team Explorer Everywhere 更新プログラムのアーカイブ" と入力します。
  8. [アーカイブ] をクリックします。
  9. [リポジトリ アーカイブ] ダイアログ ボックスが表示されます。
  10. ダウンロードされた "TFSEclipsePlugin-UpdateSiteArchive-14.0.4.zip" ファイルを検索し、ファイル名として選択します。
  11. [開く] をクリックします。
  12. [OK] をクリックします。
  13. [インストール] ダイアログ ボックスにある機能の一覧で、Eclipse 用の Team Explorer Everywhere プラグインに相当するチェック ボックスを選択します
  14. [次へ] を 2 回クリックします。
  15. マイクロソフト ソフトウェア ライセンス条項に同意する場合は、同意してから [完了] をクリックします。
  16. 要求されたときに Eclipse を再開します。

Team Foundation Server のクロスプラットフォーム コマンドライン クライアントのインストール

Team Foundation Server 用のクロスプラットフォーム コマンドライン クライアントから多くのバージョン管理操作を実行できます。 このクライアントを使用するには、これをインストールし、シェルまたはシステム パスを構成してインストール フォルダーを含めなければなりません。

Team Foundation Server 用のクロスプラットフォーム コマンドライン クライアントのインストール方法

  1. このページのリンクを使用して TEE-CLC-14.0.4.zip をダウンロードします。
  2. クライアントを含むアーカイブ (TEE-CLC-14.0.4.zip ファイル) を解凍します。
  3. シェルまたはシステム パスを構成して、アーカイブの解凍先のフォルダーを含めます。
  4. クライアントが動作していることを確認するには、コマンド プロンプトで、tf と入力し、Enter キーを押します。
クライアントが正しくインストールされると、"Microsoft Team Explorer Everywhere Command-line Client (version 14.0.4)" で始まる出力が表示されます。

コマンドの詳細については、tf help Command と入力します。ここで、Command とは、情報を必要とするコマンドの名前です。 たとえば、tf help checkin と入力します。 ヘルプを入手するには、tf command /help または tf command -help を入力することもできます。

メモ
使用する HTTP プロキシの指定、資格情報の自動キャッシュの有効化、または差分ツールやマージ ツールの制御など、シェルで環境変数を設定することによってコマンド ライン クライアントの動作を制御できます。 設定する環境変数については、次を参照してください。 環境変数の設定 MSDN のトピック。
