Installation du plug-in Team Explorer Everywhere et du client en ligne de commande

Vous pouvez utiliser Team Explorer Everywhere et Visual Studio Team Foundation Server dans vos processus de développement avec Eclipse pour faire le suivi des éléments de travail, fournir la gestion de version et automatiser les builds. Vous devez installer le plug-in Team Explorer Everywhere pour Eclipse ou le client en ligne de commande multiplateforme pour Team Foundation Server avant de pouvoir utiliser ces fonctionnalités.

Dans cette rubrique

  • Composants requis
  • Installer le plug-in Team Explorer Everywhere pour Eclipse
  • Installation de Cross-platform Command-line Client for Team Foundation Server

Composants requis

Vous devez installer une version prise en charge d'Eclipse que vous pouvez télécharger sur le site Web : Page d'accueil d'Eclipse.org..

Configuration requise pour le système d'exploitation :

  • Windows 10 (x86 et x64)
  • Windows 8.1 (x86 et x64)
  • Windows 8 (x86, x64)
  • Windows 7 (x86, x64)
  • Linux avec GLIBC 2.3+ (x86, x86_64)
  • Mac OS X 10.8+ (Intel uniquement)

Spécifications de l'environnement de développement intégré (IDE) :

  • IDE basés sur Eclipse 4.2+ sur Linux, Mac OS X, ou Windows.

Spécifications du runtime Java :

  • Oracle Java™ 1.7+ ou IBM Java™ 1.7+ sur Linux
  • Apple Java™ 1.7+ sur Mac OS X
  • Oracle Java™ 1.7+ ou IBM Java™ 1.7+ sur Windows

Spécifications de Team Foundation Server :

  • Visual Studio Team Services
  • Visual Studio Team Foundation Server 2017
  • Visual Studio Team Foundation Server 2015
  • Visual Studio Team Foundation Server 2013
  • Visual Studio Team Foundation Server 2012

Installer le plug-in Team Explorer Everywhere pour Eclipse

Vous devez installer le plug-in avant de pouvoir l'utiliser dans vos processus de développement. Vous pouvez effectuer une installation dans Eclipse directement à partir du site de mise à jour, ou vous pouvez effectuer une installation à partir de l'archive du site de mise à jour Eclipse.

Pour installer le plug-in à partir du site de mise à jour

  1. Ouvrez Eclipse.
  2. Dans le menu ? (Aide), choisissez Installer un nouveau logiciel.
  3. La boîte de dialogue Installer s'affiche.
  4. Choisissez le bouton Ajouter.
  5. La boîte de dialogue Ajouter un référentiel s'affiche.
  6. Pour le nom, entrez "Team Explorer Everywhere"
  7. Spécifiez l'emplacement du site de mise à jour : http://dl.microsoft.com/eclipse/tfs
  8. Choisissez le bouton OK.
  9. Dans la liste des fonctionnalités de la boîte de dialogue Installer, cochez la case correspondant à Plug-in Team Explorer Everywhere pour Eclipse. Si vous ne voyez pas cette option, utilisez le menu déroulant de Travailler avec, recherchez l'URL du site de mise à jour que vous venez d'entrer et sélectionnez-la, puis cochez la case en regard du plug-in mentionné plus haut.
  10. Choisissez le bouton Suivant à deux reprises.
  11. Si vous acceptez les termes du contrat de licence logiciel Microsoft, indiquez-le, puis choisissez le bouton Terminer.
  12. Redémarrez Eclipse quand vous y êtes invité.

Pour installer le plug-in à partir de l'archive du site de mise à jour

    Remarque
    Le mécanisme standard du site de mise à jour ci-dessus est le mode d'installation recommandé. Toutefois, il nécessite une connexion à Internet et entraîne l'installation de la toute dernière version du plug-in de Microsoft. Si la machine d'installation n'est pas directement connectée à Internet, ou si l'organisation préfère qu'une version spécifique du plug-in soit installée par chaque utilisateur, la méthode suivante peut s'avérer plus appropriée.

  1. Utilisez le lien sur cette page pour télécharger TFSEclipsePlugin-UpdateSiteArchive-14.0.4.zip.
  2. Ouvrez Eclipse.
  3. Dans le menu ? (Aide), choisissez Installer un nouveau logiciel.
  4. La boîte de dialogue Installer s'affiche.
  5. Choisissez le bouton Ajouter.
  6. La boîte de dialogue Ajouter un référentiel s'affiche.
  7. Pour le nom, entrez "Archive locale de mise à jour de Team Explorer Everywhere".
  8. Choisissez le bouton Archiver.
  9. La boîte de dialogue Archive du référentiel s'affiche.
  10. Recherchez et sélectionnez le fichier "TFSEclipsePlugin-UpdateSiteArchive-14.0.4.zip" téléchargé.
  11. Choisissez le bouton Ouvrir.
  12. Choisissez le bouton OK.
  13. Dans la liste des fonctionnalités de la boîte de dialogue Installer, cochez la case correspondant au Plug-in Team Explorer Everywhere pour Eclipse
  14. Choisissez le bouton Suivant à deux reprises.
  15. Si vous acceptez les termes du contrat de licence logiciel Microsoft, indiquez-le, puis choisissez le bouton Terminer.
  16. Redémarrez Eclipse quand vous y êtes invité.

Installer le client de ligne de commande multiplateforme pour Team Foundation Server

Vous pouvez effectuer de nombreuses opérations de contrôle de version à partir du client de ligne de commande multiplateforme pour Team Foundation Server. Pour pouvoir utiliser ce client, vous devez d'abord l'installer, puis configurer votre shell ou votre chemin d'accès système pour inclure le dossier d'installation.

Pour installer le client de ligne de commande multiplateforme pour Team Foundation Server

  1. Utilisez le lien sur cette page pour télécharger TEE-CLC-14.0.4.zip.
  2. Décompressez l'archive (fichier TEE-CLC-14.0.4.zip) qui contient le client.
  3. Configurez votre shell ou votre chemin d'accès système pour inclure le dossier dans lequel vous avez décompressé l'archive.
  4. Pour vérifier si le client fonctionne, à l'invite de commandes, tapez tf, puis appuyez sur la touche ENTRÉE.
Si le client est correctement installé, la sortie s'affiche, en commençant par "Client de ligne de commande Microsoft Team Explorer Everywhere (version 14.0.4)".

Pour plus d'informations sur l'une des commandes, tapez tf help Commande, où Commande représente le nom de la commande pour laquelle vous souhaitez des informations. Par exemple, tapez tf help checkin. Vous pouvez également taper tf command /help ou tf command -help pour obtenir de l'aide.

Remarque
Vous pouvez contrôler le comportement du client de ligne de commande en définissant des variables d'environnement dans votre shell, par exemple en spécifiant l'utilisation d'un proxy HTTP. Par ailleurs, vous pouvez activer la mise en cache automatique de vos informations d'identification, ou contrôler votre outil Diff/de fusion. Pour plus d'informations sur les variables d'environnement à définir, voir Définir les variables d'environnement la rubrique sur MSDN.
